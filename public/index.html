Appointments 5

Users working in this company can perform some CRUD operations, such as

  • get the list of appointments
  • get single appointment information by appointment id
  • update appointment date or status
  • delete an appointment with the given id

    • Note:
    Only the admin can perform the above operations overall appointments.

    Individual users are restricted with their appointments.

    While you get the appointment list you can perform querying in the format: ?year=yyyy&month=mm&date=dd

    You can sort the list by adding sort=ASC or sort=DESC. Default sort is in descending order.

    raw

    Create Appointment  |  POST   HOST_URL/api/v1/appointments

    Body
    { "address" : "CM20EL", "date" : "2021-12-19", "first_name" : "customer", "last_name" : "customer", "email" : "customer@email.com", "phone" : "+90-123-456-7899" }

    Delete Appointment  |  DELETE   HOST_URL/api/v1/appointments/{:id}

    Get All Appointments  |  GET   HOST_URL/api/v1/appointments/

    Get Single Appointment  |  GET   HOST_URL/api/v1/appointments/{:id}

    raw

    Update Appointment  |  PATCH   HOST_URL/api/v1/appointments/{:id}

    Description

    You can only update the appointment status or date.



    Body
    { "date": "2021-12-16" }

    Auth 3

    Anyone working in this company can do register, login, and logout operations to this API system.

    Note :
    First registered user automatically has an admin role. The others have a role as users.

    raw

    Login User  |  POST   HOST_URL/api/v1/auth/login

    Body
    { "email": "john@email.com", "password" : "secret" }

    Logout User  |  GET   HOST_URL/api/v1/auth/logout

    raw

    Register User  |  POST   HOST_URL/api/v1/auth/register

    Body
    { "name": "john doe", "email": "john@email.com", "password" : "secret" }

    Contacts 4

    CRUD operations endpoint, it is accessible by only admin.

    • manually create a customer
    • get the list of all customers contact information
    • get single customer contact information by contact id
    • update customer contact information

    raw

    Create Contact  |  POST   HOST_URL/api/v1/contacts

    Body

    Get All Contacts  |  GET   HOST_URL/api/v1/contacts

    Get Single Contact  |  GET   HOST_URL/api/v1/contacts/{:id}

    raw

    Update Contact  |  PUT   HOST_URL/api/v1/contacts/{:id}

    Body
    { "first_name" : "firstname", "last_name" : "lastname", "email" : "test-customer@email.com", "phone" : "+90-123-456-7890" }

    Users 4

    Users working in this company can perform some CRUD operations, such as

    1. Only admin can perform:
      • get the list of all users
    2. Only admin and the user who is logged in can perform
      • get single users information by user id
      • update user information
      • update user password

    Note:
    No one can see or change the others information except admin.

    Get All Users  |  GET   HOST_URL/api/v1/users

    Get Single User  |  GET   HOST_URL/api/v1/users/{:id}

    raw

    Update User  |  PATCH   HOST_URL/api/v1/users/updateUser

    Body
    { "name": "new john doe", "email": "newjohn@email.com" }
    raw

    Update User Password  |  PATCH   HOST_URL/api/v1/users/updateUserPassword

    Body
    { "oldPassword" : "secret", "newPassword" : "newsecret" }
